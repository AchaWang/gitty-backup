🛡️ 三大智慧同步與安全備份模式

根據你的金庫潔癖與安全需求，隨意切換最適合的 Rclone 傳輸策略

🟢 增量複製模式 rclone copy

最保守與放心的安全模式！ 只會將來源端新增或有修改的檔案複製到目的地硬碟。如果目的地原本就存在某些舊檔案，即使你在來源端把它刪除了，目的地硬碟也絕對不會刪除任何舊檔。

rclone copy target_dir remote_path --filter-from rclone_backup_rules.txt -v --stats 2s

⚠️ 嚴格鏡像模式 rclone sync --delete-excluded

極致潔癖與強迫症專屬！ 確保目的地硬碟與你現在過濾後的專案結構 100% 鏡像吻合。如果你刪除了檔案，或新加入了 `.gitignore` 規則（如排除 `.vs`），目的地硬碟裡對應的舊檔案也將被乾脆清除！

rclone sync target_dir remote_path --filter-from rclone_backup_rules.txt --delete-excluded -v

🛡️ 純淨鏡像 + 歷史時光機封存 rclone sync --backup-dir

👑 本專案最強大的推薦金庫模式！ 在享受 100% 嚴格鏡像同步的同時，把目的地即將被覆寫或刪除的舊版本檔案，自動遷移並按照日期編號保存到 `_Archive/yyyy-MM-dd_HHmmss/` 資料夾中。既能隨時回溯舊版代碼，又能維持目的地主目錄的絕對純淨！

rclone sync target_dir remote_path --filter-from rclone_backup_rules.txt --delete-excluded --backup-dir "Remote:_Archive/2026-07-15_235830" -v

🏛️ 專案收斂架構與一鍵建置

完全零散落的精簡目錄設計，支援本地或 CLI 快速編譯

Gitty-Backup/
├── GittyBackup/                      <-- C# WPF 現代化主控制中心
│   ├── GittyBackup.csproj            <-- .NET 10.0 WPF 專案設定檔
│   ├── MainWindow.xaml / .cs         <-- 雙向滾動日誌與後端邏輯
│   └── Scripts/                      <-- 🔥 唯一的 PowerShell 過濾引擎金庫
│       └── RcloneRuleManager.ps1     <-- 建置時 MSBuild 自動精準同步輸出
└── Standalone_Release/               <-- 🚀 125MB 便攜免安裝發布金庫
    ├── GittyBackup.exe               <-- 雙擊即開，自帶 WPF 與 .NET 引擎
    └── Scripts/RcloneRuleManager.ps1 <-- 自動對齊最新版本腳本

📦 一鍵打包免安裝單檔便攜版 (`Self-Contained Portable`)

cd GittyBackup && dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o "../Standalone_Release"
